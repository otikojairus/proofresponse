# Page Specification Template

Use this format when asking IRIS to generate a new page in the same style as the existing site.

## Route Type
- `local-service-page`
- `service-hub-page`
- `custom-landing-page`

## Target Route
- Example: `/ontario/toronto/water-damage`

## Reference Page
- File path or route that IRIS should mimic most closely
- Example: `app/[state]/[city]/[service]/page.tsx`

## Page Goal
- Example: Generate a local water damage page for Toronto that matches the current emergency-restoration style and improves local search coverage.

## Primary Topic
- Example: Water Damage Restoration

## Location
- State or province:
- City:

## Primary Keyword
- Example: `water damage restoration toronto`

## Secondary Keywords
- Example: `emergency water removal toronto`
- Example: `flood cleanup toronto`
- Example: `basement flooding cleanup toronto`

## Required Sections
- Hero
- Trust cards
- Local factors
- Scenario cards
- Process or coordination section
- FAQ
- Closing CTA

## Tone
- Example: Calm, credible, urgent, non-spammy, homeowner-friendly

## CTA Requirements
- Example: Include emergency call CTA with phone number

## Internal Links To Include
- Example: `/services/water-damage`
- Example: `/services`

## Notes
- Mention specific neighborhoods only if provided
- Keep keyword use natural
- Preserve the existing `iris-*` visual system
